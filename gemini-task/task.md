Bạn là 1 kỹ sư phần mềm senior. Nhiệm vụ: tổ chức lại thư mục `docs/` trong repo quản lý gia phả (FE: Vue3 + Vuetify, BE: ASP.NET Core).  
Repo này không dùng Jira hay GitHub Projects, nên mọi tài liệu về dev + project management phải nằm trong repo.  

Yêu cầu:  
1. Tạo cấu trúc `docs/` gồm 2 nhóm:  
   - Engineering docs: architecture, frontend, backend, API reference, data model, testing, security, development guide.  
   - Project docs: backlog, sprints, test cases, release notes, roadmap, team.  

2. Với mỗi file, hãy tạo **template Markdown chi tiết** (chỉ mục, section cần có, gợi ý nội dung).  
   - `project/backlog.md` cần có ToDo / In Progress / Done.  
   - `project/test-cases.md` cần có format test case ID, steps, expected result.  
   - `project/release-notes.md` cần có version, date, changes, known issues.  
   - `project/roadmap.md` cần có short-term / mid-term / long-term goals.  

3. Tạo `docs/README.md` để giới thiệu & link đến toàn bộ docs.  

4. Viết bằng tiếng việt, ngắn gọn, dễ đọc, giống phong cách Google hoặc open-source project docs.
