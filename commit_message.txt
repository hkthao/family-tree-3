refactor(event): Align event.store with member.store and fix EventListView pagination

This commit refactors the `event.store` to match the style and patterns of `member.store`,
improving consistency and maintainability.

Key changes include:
- `event.store.ts`:
  - Renamed `list.filter` to `list.filters`.
  - Added `list.sortBy` for consistent sorting management.
  - Removed the `IdCache` caching mechanism and all related logic.
  - Modified `setPage`, `setItemsPerPage`, and `setSortBy` actions to only update state.
  - Introduced a new `setListOptions` action to centralize pagination and sorting updates, which then triggers `_loadItems`.
  - Standardized `updateItem` action to return `Promise<Result<Event, ApiError>>`.

- `EventListView.vue`:
  - Updated to use the new `eventStore.setListOptions` action for handling pagination and sorting.
  - `fetchEventsData` now uses `eventStore.list.filters`.
  - Corrected pagination flow to resolve "cannot go to page 2" and double request issues by ensuring the initial data load is correctly triggered and itemsPerPage is managed via the store.
  - Removed unused `DEFAULT_ITEMS_PER_PAGE` import.

- `EventList.vue`:
  - Standardized event emissions for 'view', 'edit', and 'delete' to pass only the event's ID.
  - Corrected `sortBy` type in `loadEvents` function.

- `api.event.service.ts`:
  - Updated `loadItems` method to accept `sortBy` parameter.
  - Applied formatting cleanup.

- `event.service.interface.ts`:
  - Updated `loadItems` method signature to include `sortBy` parameter.

- `EventCalendar.vue`:
  - Changed `eventStore.list.filter` to `eventStore.list.filters`.

- `RelationshipDetailView.vue`:
  - Modified watch to use `props.relationshipId` instead of `route.params.id`.

- `apps/admin/package.json`:
  - Updated `vue-tsc` and `@vue/tsconfig` versions.

- `apps/admin/tsconfig.node.json`:
  - Removed `extends` and explicitly defined `target` and `lib` for TypeScript compatibility.
